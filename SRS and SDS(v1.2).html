  • Introduction

    • 1.1 Purpose

    This is the software Requirement Specification for e-commerce site named "Bazar Kori". Our site will allow customers to look up products in batch via Voice Search. It will allow for the update of catalog and inventory.

    1.2 Business Case and Relevance to Gartner's Prediction

    According to Gartner's top strategic predictions voice and visual search will contribute to 30% commerce revenue by 2021. Our site will allow customers to look up products in batch via Voice Search.

    Site will have responsive admin site as well as simple UI for updating catalogs to make sure staff of non IT background can have access. Gartner's predictions say that by 2021, 40% of IT job staff will have versatile backgrounds.

    1.3 Document Conventions

     

    The document follows IEEE SRS writing convention. The document was written in font Arial.

     

    1.3 Intended Audience and Reading Suggestions

    This document will be used by team members during design and implementation stage, by catalog keepers for updating inventory, by testers ensuring testing standards and by user for clarification.

    1.4 Project Scope

    *The site will provide the user the option to search via voice to remove the need to typing.

    *Streamlining the contemporary single search process used in many sites to offer multiple search via voice option.

    *Users Interface will be simple and intuitive in order to have great user experience.

    *Site will be responsive will being viewed on screens of different sizes.

    *Users can register and login via their social media account(s).

     

    1.4 References

    We took reference for the format from IEEE SRS documentation.


    1. Overall Description

    2.1 Product Perspective

    Our site will have the basic functionality of most e-commerce sites with the added option of performing voice search as well as multiple search at a time. It will widen the current e-commerce user/customers space by allowing access to people who cannot type in search options.

    2.2 Product Functions

    *Adding items to cart via multiple voice search as well as managers to update catalogs and admin to maintain the site.

    2.3 Classes and Characteristics

    It includes the people who registered account and will purchase the things. It will save the attributes of product sale.

    Guest Shopper

    It includes the people will purchase the things without registering. It will save the attributes of product sale.

     

    2.4 Operating Environment

    *Site will operate on most browsers such as Chrome, Firefox, Safari and Edge.

    *RAM should be more than 512MB.

    *Site will operate on Windows, macOS and Linux operating systems.

    2.5 Design and Implementation Constraints

    *Product must be ready for production by deadline.

    *Developers must strictly follow and meet the requirements specifications mentioned in this document.

    *We will use Apache server to serve web pages to user.

    *We will use Postgre Database.

    *Site will built using Django web development framework.

    2.6 User Documentation

    *This document will be freely available for all types of users to view and refer to.

    *User documentation will be available in the site.

    *Further assistance regarding user documentation will be addressed via email.

    2.7 Assumptions and Dependencies

    *Internet speed dictates page loading speed.

    *Server must be active at all times.

    1. System Features

    4.1 System Feature 1

    Voice Search

    4.1.1 Description and Priority

    Users will be able to search for and buy products using voice input which will be provided by Google API. It is one the most important features of the site.

    The Model-View-Controller architecture works by having controllers as as mediators between the models(the backend) and views(the frontend). The MVT framework adapted by django is fully MVC compliant but with different naming conventions.
